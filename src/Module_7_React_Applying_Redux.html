Q1: What is Redux?

Ans : 

        Redux is a predictable state management library for JavaScript applications, commonly used with frameworks like React. 
        It consist of mainly 
        * Store: The central repository where the application state is stored. It acts as the single source of truth for the application's data.
        * Actions: Plain objects that describe an event that has occurred in the application. 
        Each action has a type property that indicates the type of action being performed, and it may include additional data (payload).
        * Reducers: Functions that take the current state and an action as arguments and return a new state. 
        They determine how the state changes in response to actions.
        * Dispatch: The method used to send actions to the store. This is the only way to trigger a state change in Redux.

Q2: What is Redux Thunk used for?

Ans : 

   
            Redux Thunk is a middleware for Redux that allows you to write action creators as functions instead of plain objects.
            Redux Thunk is a middleware that extends the functionality of Redux by allowing you to write action creators as functions instead of objects.

Q3: What is Pure Component? When to use Pure Component over Component?

Ans : 

        A Pure Component in React is a type of component that extends React.PureComponent. 
        It is designed to optimize performance by preventing unnecessary re-renders. 
        A Pure Component only re-renders when its props or state change, which is determined through a shallow comparison

Q4: What is the second argument that can optionally be passed tosetState and what is its purpose?

Ans : 

            The second argument that can optionally be passed to setState in React is a callback function. 
            This function is executed immediately after the state has been updated and the component has re-rendered.
